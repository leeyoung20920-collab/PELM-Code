# PELM-Code
Physics-Constrained Ensemble Learning Method/物理约束下的集成学习方法（潮汐计算）
# README

## 项目简介

本代码用于对物理潮汐模型或调和分析方法得到的潮位预测结果进行机器学习订正。程序提供了一个图形化界面，用户可以导入训练潮位数据、实测潮位数据和物理潮位数据，并利用树模型学习物理预测结果与实测潮位之间的偏差关系，从而生成订正后的潮位结果。

该代码的基本思想是：不直接用机器学习模型替代物理模型，而是以物理潮位预测结果为基础输入，在其基础上构建周期特征、滞后特征、滑动统计特征和差分特征，再使用机器学习模型进行误差订正。当前程序支持 Random Forest、Extra Trees 和 Gradient Boosting 三种树模型，并可以输出订正结果及模型评价指标。

需要注意的是，虽然文件名中包含 PELM，但当前代码实际实现的是基于树模型的物理约束机器学习订正流程，而不是传统意义上的极限学习机模型。

## 输入数据说明

程序需要输入三个 CSV 文件：

1. 训练潮位数据；
2. 实测潮位数据；
3. 物理潮位数据。

三个文件都必须包含时间列。程序通过 `Time` 字段对不同数据进行匹配和合并，因此三个文件中的时间格式、时间间隔和时间戳必须保持一致。若时间无法正确对齐，程序将无法正常构建训练集或评价集。

## 1. 训练潮位数据

训练潮位数据用于训练机器学习订正模型。该文件应包含有限长度的实测潮位序列。

推荐格式如下：

Time,Tide
2020-01-01 00:00:00,1.235
2020-01-01 01:00:00,1.318
2020-01-01 02:00:00,1.402

其中：

- `Time` 为时间字段；
- 第二列为训练使用的实测潮位值；
- 第二列名称可以不是 `Tide`，程序会自动将其作为训练标签处理；
- 训练数据中的时间必须能够与物理潮位数据中的时间对应。

## 2. 实测潮位数据

实测潮位数据用于评价物理模型结果和机器学习订正结果的精度。

推荐格式如下：

Time,Tide_obs
2020-01-01 00:00:00,1.235
2020-01-01 01:00:00,1.318
2020-01-01 02:00:00,1.402

其中：

- `Time` 为时间字段；
- 第二列为实测潮位值；
- 该文件用于计算 RMSE、MAE、R² 和相对于物理模型的改进百分比；
- 建议第二列不要直接命名为 `Observed`，以避免程序内部合并时产生列名冲突；
- 推荐使用 `Tide_obs`、`WaterLevel` 或 `Observed_Tide` 等列名。

## 3. 物理潮位数据

物理潮位数据是机器学习订正的基础输入。该数据可以来自调和分析模型、S_TIDE、数值潮汐模型或其他物理潮位预测方法。

推荐格式如下：

Time,Physical
2020-01-01 00:00:00,1.180
2020-01-01 01:00:00,1.265
2020-01-01 02:00:00,1.350

其中：

- `Time` 为时间字段；
- `Physical` 为物理模型或调和分析得到的潮位预测值；
- 如果第二列未命名为 `Physical`，程序会自动将第二列重命名为 `Physical`；
- 物理潮位数据应尽量覆盖完整的待订正时段；
- 物理潮位数据的质量会直接影响机器学习订正结果。

## 数据格式注意事项

使用前应重点检查以下内容：

- 三个 CSV 文件都必须包含 `Time` 字段；
- 三个文件中的 `Time` 时间格式应保持一致；
- 三个文件中的时间间隔应保持一致，例如都为小时数据；
- 时间戳需要能够准确匹配；
- 不建议存在重复时间；
- 不建议存在大量缺测值；
- 潮位单位应保持一致，建议统一为 m；
- 物理潮位数据应覆盖训练数据和实测评价数据的时间范围。

推荐的时间格式为：

2020-01-01 00:00:00

## 输出结果

程序训练完成后，可以导出 Excel 文件。导出文件主要包含：

- 完整的物理潮位序列；
- 实测潮位序列；
- 最优机器学习模型订正后的潮位序列；
- 不同模型的订正结果；
- RMSE、MAE、R² 和改进百分比等评价指标。
